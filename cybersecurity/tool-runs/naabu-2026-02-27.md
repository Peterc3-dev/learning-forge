## 🔷 XRAY (Architecture Analysis)

### Two Scan Modes

**SYN Scan (default, requires root + libpcap):**
```
Send SYN packet → wait for SYN-ACK (port open) or RST (closed)
Never completes the TCP handshake → faster, lower footprint
Requires: root privileges + libpcap
Speed: very fast — thousands of ports/second
```

**CONNECT Scan (fallback, no root needed):**
```
Full TCP handshake → connection established → immediately close
Requires: no special privileges
Speed: slower — limited by OS socket limits
Use: when root isn't available or SYN is blocked
```

### Concurrency Model
```
Input (hosts from stdin / file / -host flag)
    ↓
DNS resolution (fastdialer — parallel)
    ↓
CDN check (optional — skip CDN IPs)
    ↓
Port list expansion (top-100 / top-1000 / full / custom)
    ↓
Worker pool — scans host:port pairs in parallel
    │
    ├── SYN mode: raw packet sender + pcap listener
    └── CONNECT mode: goroutine per connection attempt
    ↓
Open ports collected → output
```

Default concurrency: 1000 packets/second (SYN) or based on OS socket limits (CONNECT). Rate limiting is critical — pushing too hard triggers target-side defenses or burns through your scan budget.

### Port Lists Built In
| Flag | Count | What it covers |
|------|-------|----------------|
| `-top-ports 100` | 100 | Most common services — web, DB, SSH, FTP, etc. |
| `-top-ports 1000` | 1000 | Extended common services |
| `-p -` | 65535 | Full range — slow, noisy |
| `-p 80,443,8080` | custom | Specific ports only |
| `-p 1-1024` | 1024 | Well-known ports |

### naabu vs nmap — Key Differences

| Aspect | naabu | nmap |
|--------|-------|------|
| Speed | Very fast (purpose-built) | Slower by default |
| Fingerprinting | None — ports only | Service/version/OS detection |
| Scripting | None | NSE (powerful) |
| Ecosystem | ProjectDiscovery pipeline | Standalone |
| Output | host:port pairs (pipeable) | Verbose (harder to pipe) |
| Use case | Initial port discovery at scale | Deep investigation of specific hosts |
| CDN skip | Built-in | Manual |

**The workflow:** naabu for fast initial discovery across many hosts → nmap `-sV` on specific interesting ports for service fingerprinting.

## 🔷 FIRE (Execution)

### Simplest Command (Baseline)
```bash
naabu -host example.com
```
Scans top 100 ports. Prints open ports as `host:port`.

### From a Host List (standard recon use)
```bash
naabu -list live.txt -top-ports 100 -o ports.txt
```
Takes httpx output (live hosts), scans top 100 ports on each, writes results.

### Sample Output
```

api.example.com:80
```

Explicit CONNECT mode — works without sudo. Slower but functional.

## 🔷 BREAK (Parameter Variation)

### Extended Port Range
```bash
# Top 1000 — more coverage, slower
naabu -list live.txt -top-ports 1000 -o ports.txt

# Custom ports — target specific services
naabu -list live.txt -p 80,443,3000,8000,8080,8443,9000,9200 -o ports.txt
```

The custom list above targets web services specifically — useful when you want web attack surface only and don't care about DB, SSH, etc.

### Rate Limiting for Stealth
```bash
naabu -list live.txt -top-ports 100 -rate 100 -timeout 3000
```
`-rate 100` = 100 packets/second. Default is 1000. Lowering dramatically reduces detection risk and load on the target.

### Skip CDN Hosts
```bash
naabu -list live.txt -top-ports 100 -exclude-cdn
```
CDN IPs (Cloudflare, Akamai, Fastly, etc.) don't expose the real server — scanning them is mostly noise. `-exclude-cdn` skips them automatically. Cleaner results, fewer wasted scans.

### ICMP Host Discovery First
```bash
# Only scan hosts that respond to ICMP (alive check)
naabu -list all_subs.txt -ping -top-ports 100
```
Skips hosts that don't respond to ping before port scanning. Faster for large lists where many hosts may be dead.

### Pipe Directly into httpx
```bash
naabu -list live.txt -top-ports 100 -silent | \
  httpx -status-code -title -tech-detect -o web_ports.txt
```
Discovers open ports → immediately probes each port for HTTP/HTTPS services. This catches web services running on non-standard ports (3000, 8080, 8443, 9000) that httpx wouldn't see scanning just port 80/443.

**This is the move for bug bounty:** many devs run staging environments on non-standard ports with less hardening than production.

## 🔷 BRIDGE (Bounty Workflow Integration)

### Where naabu Fits
```
subfinder + findomain → all_subs.txt
    ↓
httpx -l all_subs.txt → live.txt          (filter live hosts, grab metadata)
    ↓
naabu -l live.txt → ports.txt             ← HERE (what's listening where?)
    ↓
httpx -l ports.txt → web_ports.txt        (probe non-standard ports for HTTP)
    ↓
ffuf (content discovery on interesting hosts + ports)
    ↓
nuclei -l web_ports.txt                   (vuln scan everything HTTP-accessible)
```

### High-Value Port Signals for Bug Bounty

| Port | Service | Why it matters |
|------|---------|----------------|
| 3000 | Node.js / Grafana | Dev servers, dashboards |
| 4443 | Alt HTTPS | Often less hardened than :443 |
| 8080 | Alt HTTP / Jenkins | Admin panels, CI/CD |
| 8443 | Alt HTTPS | Same as above |
| 8888 | Jupyter / dev | Exposed notebooks — critical |
| 9000 | Portainer / PHP-FPM | Container management |
| 9200 | Elasticsearch | Unauth data exposure |
| 27017 | MongoDB | Unauth DB (if exposed) |
| 6379 | Redis | Unauth cache (if exposed) |
| 5432 | PostgreSQL | DB on public interface |

Non-standard web ports are where lazy deployments hide. A dev spins up a service on :3000 with no auth because "it's not the main site." That's your way in.

### Full Pipeline from Scratch
```bash
# Step 1: Passive subdomain enum
{ subfinder -d target.com -silent; findomain -t target.com -q 2>/dev/null; } | \
  sort -u > all_subs.txt

# Step 2: Filter live HTTP hosts
httpx -l all_subs.txt -silent -o live.txt

# Step 3: Port scan live hosts
naabu -list live.txt -top-ports 1000 -exclude-cdn -silent -o ports.txt

# Step 4: Probe non-standard ports for HTTP
httpx -l ports.txt -silent -status-code -title -tech-detect -o web_ports.txt

# Step 5: Vuln scan
nuclei -l web_ports.txt -t ~/nuclei-templates/
```
Five commands. Full recon pipeline. Automated.

### Key Flags Reference
```
-host HOST         Single target host
-list FILE         Host list file (one per line)
-p PORTS           Specific ports or ranges (80,443 or 1-1000)
-top-ports N       Top N ports by frequency (100 or 1000)
-o FILE            Output file
-json              JSON output
-silent            Ports only — no banner (good for piping)
-rate N            Packets per second (default 1000, lower = stealthier)
-timeout N         Milliseconds before timeout (default 1000)
-connect-scan      Force CONNECT mode (no root needed)
-exclude-cdn       Skip CDN IP ranges
-ping              ICMP host discovery before scanning
-exclude-ports P   Skip specific ports
-exclude-hosts F   Skip specific hosts (file)
-nmap              Pass results to nmap for service detection
-nmap-cli STR      Custom nmap command string
```
